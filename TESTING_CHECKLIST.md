# Token 过期功能测试清单

## 测试前准备
- [ ] 确保应用已编译成功
- [ ] 确保有可用的 Movie Pilot 服务器
- [ ] 准备测试账号和密码

## 功能测试

### 1. 凭据保存功能
- [ ] 首次登录：输入自定义的 API 地址、用户名、密码
- [ ] 点击"登录"按钮，验证登录成功
- [ ] 手动登出（从设置中）
- [ ] 返回登录页，验证输入框已自动填充上次输入的凭据

### 2. 一键登录功能
- [ ] 在登录页，当有保存的凭据时，应该看到绿色的"使用上次保存的凭据登录"按钮
- [ ] 点击该按钮，验证能够直接登录（无需手动输入）

### 3. Token 过期检测
- [ ] 正常登录应用
- [ ] 模拟 Token 过期（方法见下文）
- [ ] 在应用中执行任意 API 操作（浏览、搜索等）
- [ ] 验证是否弹出"登录已过期，请重新登录"提示框
- [ ] 点击提示框的"确定"按钮
- [ ] 验证是否自动跳转到登录页

### 4. Token 过期后快速重新登录
- [ ] 继续上一步，在登录页应该看到凭据已自动填充
- [ ] 点击绿色的"使用上次保存的凭据登录"按钮
- [ ] 验证能够快速重新登录成功

### 5. 应用重启凭据持久化
- [ ] 登录应用
- [ ] 完全关闭应用（Force Quit）
- [ ] 重新启动应用
- [ ] 如果 token 仍有效，应该自动登录进入主界面
- [ ] 如果 token 已过期，应该显示登录页且凭据已填充

## 模拟 Token 过期的方法

### 方法一：修改 UserDefaults（推荐）
1. 登录后，在 Xcode 的 Debug Console 中找到打印的 access token
2. 暂停应用，在 LLDB 中执行：
   ```
   po UserDefaults.standard.set("invalid_token_12345", forKey: "accessToken")
   ```
3. 继续运行，执行任意 API 操作

### 方法二：服务器端使 Token 失效
1. 如果有服务器访问权限，手动使当前 token 失效
2. 在应用中执行 API 操作

### 方法三：等待自然过期
1. 等待 token 的有效期自然过期（可能需要较长时间）
2. 再次使用应用时会触发 token 过期流程

## 预期结果总结

✅ **Token 过期时**：
- 弹出中文提示："登录已过期，请重新登录"
- 自动登出并跳转到登录页
- 登录页自动填充之前的凭据
- 显示绿色的快速登录按钮

✅ **凭据记忆**：
- API 地址、用户名、密码在登录后自动保存
- 登出后重新进入登录页，凭据仍然填充
- 应用重启后，凭据仍然保存

✅ **一键登录**：
- 绿色按钮显示"使用上次保存的凭据登录"
- 点击后无需手动输入，直接使用保存的凭据登录
- 适用于 token 过期后快速重新登录的场景

## 已知限制

⚠️ **安全性**：
- 密码当前以明文形式存储在 UserDefaults
- 生产环境建议使用 iOS Keychain

⚠️ **Token 刷新**：
- 当前实现为完全重新登录
- 未来可考虑实现 Refresh Token 机制
